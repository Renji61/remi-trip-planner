What you can do with REMI Trip Planner

  • Create and organize trips with dates, description, cover image, and currency.
  • For new trips you own, use the optional first-trip setup wizard (name, dates, description, cover image or preset, currency, and travel defaults) until onboarding is complete.
  • See trips grouped as active, draft, or archived on your personal dashboard.
  • Invite others by email or shareable link and plan together.
  • Build a day-by-day itinerary with stops, notes, optional costs, and an interactive map (OpenStreetMap or optional Google Maps) with pins, day filters, and travel hints between stops. Add same-day commute / travel legs between two stops (transport mode, timeline order, sidebar, and calendar subscription / ICS); duplicate pairs are rejected and deleting a neighbor clears links. Optional Open in Google Maps directions between endpoints use the commute’s mode when both have coordinates, and each day can open all non-commute places as a single Google Maps route. Draft trips without start/end dates can still plan by calendar date and later remap those days when trip dates are set. Day descriptions save inline as you type, and week calendar all-day bookings now render as continuous spans across consecutive days.
  • Turn location lookup on or off when adding places; geocoding and place suggestions can be cached server-side for faster repeat lookups.
  • Track spending against a budget, with categories, Expenses and Group expenses views per trip, and splits that stay consistent when participants leave — amounts are stored in whole minor units (e.g. cents) for accurate arithmetic. Quick-add entry points use a unified expense form: save a personal expense or, when group expenses are enabled and the trip has someone to split with, toggle split with group and pick participants in the same flow.
  • When collaborators edit the same trip, optimistic concurrency can return a clear conflict instead of silently overwriting another person’s save; refresh and try again with the latest data.
  • Record accommodations, vehicle rentals, and flights with details, attachments, and per-booking status (booked vs to be booked). The main trip page includes a Booking details panel to browse stays, vehicles, and flights together with type and state filters. Saves that touch itinerary lines use atomic transactions so linked data does not end up half-updated.
  • With optional AirLabs and OpenWeatherMap API keys in site settings, flight forms gain airport and airline suggestions (with server-side caching and fallbacks), and plain itinerary stops with coordinates can show a compact same-day weather summary when the day falls in the forecast window. Saved OpenWeatherMap keys can be copied quickly from settings when rotating credentials across environments.
  • Open Trip Documents from the trip sidebar to upload files, search and filter every attachment in one place (including documents from stays, rentals, flights, and group expenses), with a configurable max file size in app settings and upload checks for safer file types.
  • Use reminder checklists so nothing important is forgotten, with shared mobile quick-add flyouts available from trip pages and subpages (notes, flights, stays, rentals, documents, expenses, group expenses, and settings). Unbooked flights can keep a Trip Bookings checklist line in sync (for example Book: … with airports), updating when the flight changes and marking done when you mark the flight booked, or you can dismiss the link.
  • Maintain a personal Notes & Checklists library on the dashboard (/notes-checklists): reusable notes and checklist templates with search, archive, and trash.
  • Import selected library items into a trip from Trip Notes (/trips/…/notes/import); the app remembers what was already imported so you can add templates once and reuse them across trips.
  • Pages that use HTMX load a vendored script from this install (not a third-party CDN); shared app.css and app.js use one built-in version query so browsers pick up updates together.
  • Customize each trip: which sections appear, labels, layout order, 12h/24h clock, and DD/MM/YYYY or MM/DD/YYYY date display.
  • Add custom links in the trip sidebar for quick references.
  • Stay in sync while a trip is open: the app can use a live event stream of changes so lists and panels refresh without reloading the whole page.
  • Export a JSON snapshot of your visible data from Profile (no password material; app secrets redacted) and use optional backup scripts or Compose profile for SQLite snapshots.
  • Use the About page to see your installed version and check for updates on self-hosted installs: the app compares against the latest stable version from GitHub Releases and version tags (not only formal release posts), and can tell you if you are on a build newer than the latest published tag.
  • Install as an app (PWA) for a more app-like experience on your device.
